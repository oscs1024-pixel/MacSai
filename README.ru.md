## Что такое Mac Sai?

Mac Sai — **бесплатное приложение для macOS с открытым исходным кодом**, которое очищает ненужные файлы, удаляет вредоносное ПО, оптимизирует производительность, полностью удаляет приложения и наглядно показывает использование диска — всё это в едином красивом интерфейсе. Оно повторяет все основные возможности CleanMyMac, оставаясь полностью прозрачным продуктом, который развивается сообществом.

**Без подписки. Без телеметрии. Без рекламы. Только чистый Mac.**

**Доступно на английском, упрощённом китайском (简体中文) и русском языках.** Язык можно в любой момент изменить в разделе **Настройки → Язык интерфейса**; по умолчанию используется системный язык.

### Виджет в строке меню

<p align="center">
  <img src="assets/menu_bar.png" width="300" alt="Виджет Mac Sai в строке меню" />
</p>

Виджет в строке меню с эффектом матового стекла позволяет одним нажатием проверить основные показатели Mac. Он работает как отдельный процесс, запускается при входе в систему и включается или выключается на боковой панели приложения. Чтобы быстро проверить состояние системы, открывать главное окно не нужно.

- **Кольцевые индикаторы в реальном времени** — загрузка ЦП, нагрузка на память, использование диска и заряд аккумулятора в сетке 2 × 2 (`host_processor_info`, `vm_statistics64`, ёмкость APFS, источник питания IOKit) с цветовой шкалой от зелёного через янтарный к красному
- **Настраиваемый показатель в строке меню** — выберите свободное место на диске, загрузку GPU, использование памяти или температуру аккумулятора; выбранный показатель сохраняется, а для недоступных датчиков отображается `--`
- **Сеть, время работы и подкачка** — скорость отправки и получения данных в реальном времени, время работы системы и использование файла подкачки
- **Рекомендации** — полезные подсказки, которые можно скрыть («Пользовательские кэши выросли до 2,52 ГБ — запустите очистку системного мусора»), с действиями в одно нажатие; после скрытия не показываются в течение 30 дней
- **Состояние защиты** — время последнего сканирования на вредоносное ПО и количество угроз с цветовым обозначением актуальности проверки
- **Подключённые устройства** — обзор внешних томов (со свободным местом) и внешних дисплеев
- **Предупреждения о состоянии системы** — фоновые уведомления при критически малом свободном месте на диске или длительной высокой нагрузке на память (с ограничением частоты, включаются пользователем)
- **Переход к приложению одним нажатием** — быстрое открытие Mac Sai

### Модель безопасности

Mac Sai спроектирован так, чтобы **никогда не приводить к потере данных**:

- **Список защищённых путей** — `/System`, `/usr`, `/bin`, `/sbin` и системные приложения Apple недоступны для изменения
- **Каноникализация firmlink-путей в macOS** — пути `/var`↔`/private/var`, `/tmp`↔`/private/tmp` и `/etc`↔`/private/etc` приводятся к единой канонической форме, чтобы проверка перенаправления символических ссылок не давала ложных срабатываний на допустимых системных путях
- **Предварительная проверка возможности очистки** — объекты, которые текущий процесс не может переместить в Корзину (дочерние объекты системных кэшей, принадлежащие root, и каталоги в `~/Library/Caches/com.apple.*`, защищённые механизмом Data Vault macOS), исключаются ещё во время сканирования и не отображаются в интерфейсе как доступные для очистки
- **Перемещение в Корзину по умолчанию** — по умолчанию все удаляемые объекты перемещаются в Корзину
- **Режим пробного запуска** — предварительный просмотр удаляемых объектов без каких-либо изменений
- **Защита от TOCTOU** — символические ссылки повторно разрешаются непосредственно перед удалением
- **Пакетная очистка** — при выборе 50 тыс. объектов и более приложение запрашивает подтверждение; механизм разбивает работу на пакеты по 5 тыс. объектов и проверяет `Task.isCancelled` между пакетами, чтобы быстро реагировать на отмену
- **Рекурсивный подсчёт байтов** — размер каталога рассчитывается обходом его содержимого, а не вызовом stat, поэтому значение «Освобождено X» на экране завершения соответствует действительности
- **Политика безопасной очистки остатков приложений** — очистка ограничена остаточными файлами в кэшах и журналах
- **Просмотр журнала действий в приложении** — каждая ошибка очистки записывается вместе с полным путём; на экране после очистки есть кнопка «Открыть журнал», которая открывает окно журнала с фильтром только по ошибкам и возможностью скопировать текст в буфер обмена, чтобы дословно вставить его в отчёт об ошибке. Записи старше 30 дней удаляются автоматически
- **Контролируемый ядром шлюз привилегий XPC** — привилегированный помощник использует `NSXPCListener.setCodeSigningRequirement` (macOS 13+), поэтому ядро самостоятельно отклоняет подключения от процессов, подпись кода которых не соответствует идентификатору основного приложения и Team ID команды разработчика

## Установка

### Homebrew (рекомендуемый способ — одна команда, без предупреждений)

Mac Sai находится в официальном репозитории cask-пакетов Homebrew, поэтому добавлять отдельный tap не нужно:

```bash
brew install --cask mac-sai
```

Приложение Mac Sai нотариально заверено Apple, поэтому оно запускается из Spotlight или папки «Программы» без предупреждений, запуска через контекстное меню и дополнительных команд.

> Устанавливали приложение из прежнего tap `iliyami/macsai`? Теперь, когда Mac Sai входит в официальный репозиторий cask-пакетов Homebrew, tap можно удалить: `brew untap iliyami/macsai` (это не повлияет на установленное приложение и последующие вызовы `brew upgrade`).

### Установка одной командой

```bash
curl -fsSL https://raw.githubusercontent.com/iliyami/MacSai/main/scripts/install.sh | bash
```

Эта команда скачивает последний DMG-образ и устанавливает приложение в `/Applications`.

### Загрузка DMG

Скачайте последний DMG-образ со страницы [Releases](https://github.com/iliyami/MacSai/releases/latest) и перетащите Mac Sai в папку «Программы». Приложение Mac Sai нотариально заверено Apple, поэтому оно запускается обычным способом, без предупреждений Gatekeeper и дополнительных команд.

### Сборка из исходного кода

```bash
git clone https://github.com/iliyami/MacSai.git
cd MacSai
swift build
swift test                     # запустить 486 тестов
bash scripts/build-dmg.sh      # собрать локальный DMG без подписи
```

### Предоставление полного доступа к диску

Некоторым модулям («Почтовые вложения», «Конфиденциальность» и «Удаление угроз») для сканирования защищённых областей требуется полный доступ к диску:

1. Откройте **Системные настройки → Конфиденциальность и безопасность → Полный доступ к диску**
2. Нажмите **+** и добавьте **Mac Sai.app** из папки «Программы»
3. Перезапустите Mac Sai

### Удаление

Если вы установили приложение с помощью Homebrew, удалите его вместе со всеми вспомогательными файлами одной командой:

```bash
brew uninstall --zap --cask mac-sai
```

Если вы установили приложение из DMG или вручную, запустите сценарий удаления (он также поддерживает установку через Homebrew):

```bash
curl -fsSL https://raw.githubusercontent.com/iliyami/MacSai/main/scripts/uninstall.sh | bash
```

Оба способа удаляют Mac Sai, его настройки, кэши, журналы и базу данных из `~/Library`. Если в разделе **Системные настройки → Основные → Объекты входа и расширения** останется запись, удалите её вручную; macOS окончательно уберёт её после удаления приложения.

## Подпись и нотариальное заверение: почему приложению можно доверять

Приложение Mac Sai подписано сертификатом Apple **Developer ID** и **нотариально заверено Apple**. Для приложения очистки это особенно важно, поскольку вы предоставляете ему расширенный доступ к своим файлам. Вы вправе быть уверены, что на вашем Mac запускается именно наша подлинная сборка, в которую никто не вмешивался.

Вот какие гарантии обеспечивает сам Mac, а не только наши слова:

- **Apple проверяет приложение.** Каждая версия перед выпуском отправляется в Apple и проверяется на наличие вредоносного ПО. Нотариальное заверение подтверждает, что именно эта сборка успешно прошла проверку Apple.
- **Приложение невозможно незаметно изменить.** Подпись служит криптографической печатью для каждого файла приложения. Если после подписания изменится хотя бы один байт — из-за повреждения загрузки, сетевой атаки или попытки вредоносной программы воспользоваться нашим именем, — macOS откажется открывать приложение.
- **Авторство можно проверить.** Подпись связана с нашей учётной записью разработчика Apple, поэтому никто другой не сможет выпустить приложение, которое ваш Mac примет за Mac Sai.
- **Приложение просто работает.** Без предупреждений Gatekeeper, запуска через контекстное меню и команд в Терминале. Установите и запустите его, как любое другое приложение, которому доверяете.

В сочетании с полностью открытым исходным кодом это образует проверяемую цепочку доверия: код общедоступен, мы подписываем каждую версию, Apple проверяет её, а ваш Mac повторно проверяет подпись при каждом запуске приложения.

Для сопровождающих проекта: порядок сборки, подписания и нотариального заверения версий описан в файле [`docs/RELEASING.md`](docs/RELEASING.md).

```
Sources/
├── MacClean/
│   ├── App/                    # Точка входа, состояние и основное представление приложения
│   ├── Core/
│   │   ├── Scanner/            # FileTreeScanner, TargetedScanner, ScanCoordinator
│   │   ├── Cleaner/            # CleaningEngine, SafetyGuard
│   │   ├── Cache/              # Слой базы данных GRDB
│   │   └── FSMonitor/          # Монитор инкрементальных изменений FSEvents
│   ├── Modules/                # 13 модулей сканирования
│   │   ├── SystemJunk/         # 16 категорий ненужных файлов
│   │   ├── Malware/            # Сканер сигнатур и мониторинг в реальном времени
│   │   ├── Uninstaller/        # Десятиуровневый механизм сопоставления приложений
│   │   ├── SpaceLens/          # Алгоритм древовидной карты с разбиением на прямоугольники
│   │   ├── Duplicates/         # Поэтапный конвейер хэширования
│   │   └── ...
│   ├── Views/                  # Представления SwiftUI (14 модулей и общие компоненты)
│   ├── ViewModels/             # Модели представления @Observable
│   ├── Services/               # PermissionManager, XPCClient
│   └── Utilities/              # Форма SuperEllipse и расширения
├── MacCleanKit/                # Общие модели, константы и протоколы
├── MacCleanHelper/             # Привилегированный помощник XPC (операции от имени root)
└── MacCleanMenu/               # Системный монитор в строке меню

Tests/                          # Набор тестов XCTest — 486 тестов
├── MacCleanTests/              # Тесты основного приложения
├── MacCleanKitTests/           # Тесты фреймворка
└── MacCleanTestSupport/        # Фикстуры (withTempHome, withFakeApp, …)
```

## Тесты

```bash
swift test
```

Набор тестов на базе XCTest охватывает:

- **`SafetyGuard`** — 24 теста на устойчивость к атакам и некорректным данным (символические ссылки, обход каталогов, нулевые байты, SIP, защищённые приложения, ограничения количества файлов и идемпотентность)
- **`CleaningEngine`** — 9 интеграционных тестов (пробный запуск, Корзина, безвозвратное удаление, обработка ошибок и журнал операций)
- **`PlistJunkFilter`** — 9 тестов, включая контракт безопасности для системного домена Apple
- Конечный автомат **`ScanCoordinator`** — сканирование, отмена, фильтрация по категориям и включение ресурсоёмких операций
- Интеграционные тесты **`TargetedScanner`** — работа с синтетическими фикстурами во временном каталоге
- **Все 16 категорий системного мусора** — декларативные описания целей сканирования и логика фильтрации для процедурных категорий (`BrokenPreferences`, `BrokenLoginItems`, `UniversalBinaries`, `DeletedUsers`)
- **`SquarifiedTreemap`** — пустой набор, один и несколько узлов, сохранение площади и свойства соотношения сторон
- **`AppMatching`** — все 10 уровней механизма сопоставления при удалении приложений
- **`DuplicateDetection`** — группы по размеру, группы по частичному и полному хэшу, устранение повторов по индексным дескрипторам
- **`MalwareSignatures`** — шаблоны имён и подозрительное содержимое агентов запуска
- **`MaintenanceTask`** — у всех 10 задач есть описания, значки и пути к исполняемым файлам
- **`FileGroup`** — группировка по размеру, типу и возрасту
- **`AppcastParser`** — разбор XML Sparkle
- **`VolumeInfo`** — расчёт использования и сравнение
- **`AppDatabase`** — операции CRUD кэша GRDB, миграции и инвалидация
- **`FSEventMonitor`** — вычисление путей, требующих инвалидации
- **`AppDiscovery`**, **`AppPathFinder`** — smoke-тесты
- **Сквозное тестирование** — синтетическая фикстура → сканирование → результаты → цикл очистки

Инфраструктура тестирования (`Tests/MacCleanTestSupport/`) предоставляет `withTempHome`, `withFakeApp`, `withFakePlist` и другие вспомогательные фикстуры, поэтому тесты всегда дают воспроизводимые результаты и никогда не затрагивают реальную домашнюю папку пользователя.

Целевое покрытие: **не менее 85% в целом**, **100% для `SafetyGuard` и `CleaningEngine`** (критически важных файлов). Полный план приведён в документе [`docs/TESTING.md`](docs/TESTING.md).

## Безопасность

Mac Sai уделяет безопасности особое внимание:

- **Без телеметрии и аналитики.** Единственный сетевой запрос — необязательная проверка обновлений (один запрос к API GitHub Releases), которую можно отключить в настройках
- **По умолчанию без повышенных привилегий** — помощник XPC активируется только для задач обслуживания
- **Проверка подписи кода** — помощник XPC проверяет подлинность вызывающего процесса
- **Защищённые пути** — более 27 системных приложений Apple и все пути, защищённые SIP, занесены в список блокировки
- **Открытый исходный код** — можно проверить каждую строку кода

### Контрольный список аудита безопасности

- [x] Нет способов внедрить команды (все аргументы Process заданы константами)
- [x] Невозможно удалить произвольный файл (SafetyGuard проверяет каждый путь)
- [x] Защита от гонок TOCTOU (повторное разрешение символических ссылок перед удалением)
- [x] Ограничение количества файлов в операции (не более 10 000 файлов)
- [x] Проверка вызывающего процесса XPC по подписи кода
- [x] В исходном коде нет секретов и учётных данных
- [x] По умолчанию файлы перемещаются в Корзину (их можно восстановить)
- [x] Журнал аудита операций (записывается каждое действие)

## Участие в разработке

Мы приветствуем вклад в проект! Перед отправкой PR ознакомьтесь с [руководством для участников](CONTRIBUTING.md).

### Быстрый старт

1. Создайте форк репозитория
2. Создайте ветку для новой функции (`git checkout -b feature/amazing-feature`)
3. Внесите изменения
4. Запустите тесты (`swift test`)
5. Создайте коммит (`git commit -m 'Add amazing feature'`)
6. Отправьте ветку (`git push origin feature/amazing-feature`)
7. Откройте Pull Request
